# 非機能設計書

**プロジェクト名**: ぐんまこどもイベント（gunma-kids-events）  
**バージョン**: 1.0.0  
**作成日**: 2026年5月16日

---

## 1. パフォーマンス

### 1.1 ビルド

- ビルドツール: Vite 5
- `vite build` により HTML / CSS / JS を静的ファイルに変換して出力
- `base: './'` 設定で相対パス参照（サブディレクトリ・file:// デプロイ対応）

### 1.2 データ読み込み

- イベントデータ（EVENTS）・情報源データ（SOURCES）・祝日データはビルド時に JS バンドルに静的に埋め込まれる
- ランタイムでの API リクエストなし（スクレイパー実行はビルド前の手動操作）

### 1.3 描画

- Vue 3 の仮想 DOM により差分更新
- フィルター・ソートはすべてクライアントサイドのインメモリ処理（リクエスト発生なし）

---

## 2. 可用性・デプロイ

### 2.1 ホスティング要件

- 静的ファイル（HTML / CSS / JS）のみをホストできれば動作する
- サーバーサイドの処理・データベースは不要

### 2.2 ルーティング

- Vue Router の Hash モード（`createWebHashHistory`）を使用
- URL に `#` を含む形式（例: `index.html#/events`）
- サーバーの URL リライト設定が不要

### 2.3 スクレイパー実行

- スクレイパー（`node scraper/scrape.js`）は手動で実行する
- 実行後、`src/data/events.js` が更新されるため、その後 `npm run build` が必要

---

## 3. セキュリティ

### 3.1 外部リンク

- 外部サイトへのリンクにはすべて `rel="noopener noreferrer"` を付与（`EventModal.vue`）
- `target="_blank"` 使用箇所: 公式サイト・LINE 共有・Google カレンダー・情報源

### 3.2 XSS 対策

- Vue 3 はデフォルトでテンプレート内のデータを HTML エスケープする
- `v-html` ディレクティブを使用している箇所はない

### 3.3 localStorage

- 保存するのはイベント ID の数値配列のみ（キー: `gunma_favs`）
- 読み込み時に `JSON.parse` が失敗した場合は `[]` にフォールバック（クラッシュしない）

### 3.4 スクレイパーのセキュリティ

- User-Agent: `Mozilla/5.0 (compatible; GunmaEventsBot/1.0)`（一部の取得元は Windows UA）
- 取得対象は既知の公的サイト・施設サイトのみ
- `loadManualEvents()` 内で既知ファイル（`src/data/events.js`）のみを `eval` する
- HTTP タイムアウト: 10,000ms（各リクエスト）
- ページネーション上限: 最大 15 ページ（ぐんラボ！）
- レートリミット対策: ぐんラボ！でのページ間に 1,000ms 待機

## 4. アクセシビリティ

### 4.1 実装済み

- ハンバーガーメニューボタンに `aria-label="メニュー"` 付与
- モーダル閉じるボタンに `aria-label="閉じる"` 付与
- `rel="noopener noreferrer"` で外部リンクの安全性確保

### 4.2 未実装

- `alt` テキストなし（画像要素なし、絵文字をアイコンとして使用）
- `role` / `aria-*` の詳細な ARIA 属性は最小限の実装

## 5. レスポンシブデザイン

### 5.1 ブレークポイント

CSS メディアクエリによる 2 段階のレスポンシブ対応。

| ブレークポイント   | 適用範囲           |
| ------------------ | ------------------ |
| `max-width: 768px` | タブレット以下     |
| `max-width: 480px` | スマートフォン以下 |

### 5.2 768px 以下での変更点

- ヘッダーナビを非表示、ハンバーガーメニューを表示
- イベントグリッドを 1 カラムに変更
- カレンダーグリッドのフォントサイズ縮小
- カレンダーのイベントドットを非表示（`.cal-event-dot { display: none }`）
- モーダルのパディング縮小

### 5.3 480px 以下での変更点

- ロゴテキストのフォントサイズ縮小
- ヒーローボタンを縦並び・全幅に変更
- ボタン最大幅 300px に制限

## 7. 保守性

### 7.1 データ管理の分離

- 手動イベント（id 1〜999）と自動収集イベント（id 10000〜99999）はファイル内で ID 範囲により区別
- スクレイパーが上書きするのは id 10000 以上の範囲のみ

### 7.2 テストカバレッジ

以下のロジック層にユニットテストが存在する:

- `useEvents.js`（全エクスポート関数）
- CalendarView の monthEvents・eventsOnDay・月ナビゲーション・daysInMonth ロジック
- EventsView のフィルター・検索・ソートロジック
- `useFavoritesStore`（全操作・localStorage 永続化・エラーハンドリング）
- `src/data/events.js` のデータ整合性（全件）
- `stableId` の冪等性・範囲

コンポーネントの DOM レンダリングテスト（`@vue/test-utils` によるマウント）は実装されていない。

### 7.3 既知の課題（テストコード記録）

| 項目          | 内容                                                                                    |
| ------------- | --------------------------------------------------------------------------------------- |
| BUG #10       | `toggleFav` の引数が文字列 ID の場合、数値 ID と `===` 比較が不一致になる（型強制なし） |
| FavoritesView | ルーター（`src/router/index.js`）に `/favorites` ルートが定義されていない               |

## 10. 地図データ

- タイルプロバイダー: OpenStreetMap（`https://tile.openstreetmap.org/`）
- Leaflet のデフォルトタイル設定を使用
- 地図表示範囲: 群馬県境界（SW: `[35.95, 138.25]`、NE: `[37.05, 139.75]`）
- ズームレベル: minZoom=10、maxZoom=14
- マーカー画像: Leaflet のデフォルト画像（Vite 環境でのパス修正適用済み）

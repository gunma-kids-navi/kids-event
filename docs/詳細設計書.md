## 4. ユーティリティ（`src/composables/useEvents.js`）

### 4.1 エクスポート一覧

| 名前                          | 種別        | 説明                                                              |
| ----------------------------- | ----------- | ----------------------------------------------------------------- |
| `TODAY`                       | `Date` 定数 | モジュール読み込み時の日時（CalendarView 初期月に使用）           |
| `parseDate(s)`                | 関数        | `YYYY-MM-DD` 文字列を `new Date(s + 'T00:00:00')` で Date 変換    |
| `formatDate(s)`               | 関数        | `YYYY-MM-DD` を `YYYY年M月D日` 形式に変換                         |
| `formatDateRange(start, end)` | 関数        | 日付範囲を日本語形式に変換（同日・同月・異月・逆転の4ケース対応） |
| `getStatus(ev)`               | 関数        | イベントのステータス文字列を返す                                  |
| `statusLabel(s)`              | 関数        | ステータス文字列 → 日本語ラベル（未知キーは `'不明'`）            |
| `statusClass(s)`              | 関数        | ステータス文字列 → CSS クラス名（未知キーは `''`）                |
| `daysLeft(ev)`                | 関数        | 今日から終了日までの残り日数（`Math.ceil`）                       |
| `daysToStart(ev)`             | 関数        | 今日から開始日までの日数（`Math.ceil`）                           |

### 4.2 `getTodayMidnight()`（内部関数）

```js
function getTodayMidnight() {
  const now = new Date();
  return new Date(now.getFullYear(), now.getMonth(), now.getDate());
}
```

`getStatus`・`daysLeft`・`daysToStart` から毎回呼び出す。今日の深夜 0 時を基準とすることで、終了日当日を `ended` ではなく `ongoing` と判定する。

### 4.3 `formatDateRange` の変換ルール

| 条件                  | 出力形式                                     |
| --------------------- | -------------------------------------------- |
| `start === end`       | `formatDate(start)`                          |
| `start > end`（逆転） | `${formatDate(start)} 〜 ${formatDate(end)}` |
| 同年・同月            | `YYYY年M月D日〜D日`                          |
| それ以外              | `${formatDate(start)} 〜 ${formatDate(end)}` |

### 4.4 `statusLabel` / `statusClass` マッピング

```js
// statusLabel
{ ongoing: '開催中', upcoming: '近日開催', ended: '終了' }

// statusClass
{ ongoing: 'badge--ongoing', upcoming: 'badge--upcoming', ended: 'badge--ended' }
```

### 5.2 `src/data/holidays.js`

`HOLIDAY_DATES`（`Set<string>`）と `isHoliday(year, month, day)` をエクスポートする。

- 対象年: 2025〜2027 年
- 参照: 内閣府「国民の祝日について」
- `month` は 0-indexed

```js
export function isHoliday(year, month, day) {
  const mm = String(month + 1).padStart(2, "0");
  const dd = String(day).padStart(2, "0");
  return HOLIDAY_DATES.has(`${year}-${mm}-${dd}`);
}
```

## 6. コンポーネント詳細

### 6.1 `App.vue`

**props / emits**: なし  
**provide**: `openModal(event)` — EventModal に表示するイベントを `modalEvent` ref にセット  
**state**: `mobileNavOpen`（boolean）、`modalEvent`（object | null）

ヘッダーナビ・モバイルハンバーガーメニュー・`<router-view />`・`<EventModal>` を含む。

### 6.2 `EventCard.vue`

**props**: `event` (Object, required)  
**emits**: `open-modal`  
**computed**:

- `status` = `getStatus(event)`
- `remainDays` = `daysLeft(event)` — `ongoing` かつ 0〜7 日のとき赤字で残り日数表示
- `startDays` = `daysToStart(event)` — `upcoming` かつ 7 日以内のとき青字で開始日表示

カード全体クリックで `$emit('open-modal', event)`。

### 6.3 `EventListItem.vue`

**props**: `event` (Object, required)  
**emits**: `open-modal`  
**computed**: `status` = `getStatus(event)`

行全体クリックで `$emit('open-modal', event)`。

### 6.4 `EventModal.vue`

**props**: `event` (Object | null)  
**emits**: `close`  
**inject**: なし（props から event を受け取る）

- `status` = `getStatus(event)`
- `lineShareUrl` = `https://social-plugins.line.me/lineit/share?url=<URLエンコードしたイベントURL>&text=<URLエンコードしたタイトル>`
- `googleCalendarUrl` = `https://calendar.google.com/calendar/render?action=TEMPLATE&text=...&dates=YYYYMMDD/YYYYMMDD&location=...&details=...` 形式（endDate は +1日して渡す）
- `copied`（boolean）— URLコピー後 2 秒間 `true`
- `copyToClipboard()` — タイトル・日程・会場・説明・URL を改行区切りで結合したテキストを `navigator.clipboard.writeText()` でコピー

### 6.5 `HomeView.vue`

**inject**: `openModal`  
**computed**:

- `ongoing` — `getStatus` が `'ongoing'` のイベント
- `upcoming` — `getStatus` が `'upcoming'` のイベント
- `areas` — EVENTS から一意エリアを抽出してソート
- `areaCounts` — エリアごとのイベント件数

**methods**: `goToArea(area)` — `/events?area=<area>` へ遷移

### 6.6 `EventsView.vue`

**inject**: `openModal`  
**state**:

- `filterStatus` ref（初期値: `'all'`）
- `filterCategory` ref（初期値: `'all'`）
- `filterArea` ref（初期値: `'all'`）
- `filterSources` ref（初期値: `[]`）— 選択中の情報源名の配列（複数選択対応）
- `searchQuery` ref（初期値: `''`）
- `sortOrder` ref（初期値: `'date-asc'`）

**URL パラメータ連携**: `onMounted` + `watch(route.query)` で `?area=xxx` を `filterArea` に反映

**フィルター適用順**: status → category → area → source → keyword

**情報源フィルター**:  
`filterSources` が空配列のときは全件表示。1件以上選択時は、選択された情報源の `matchUrl` を取得し、イベントの `url` がいずれか 1 つ以上に部分一致（OR 条件）するものに絞り込む。`toggleSource(name)` 関数で追加・削除をトグルする。

**keyword 検索対象フィールド**: title、area、venue、tags（各要素）、label、desc

**ソート**:

- `date-asc`: `parseDate(startDate)` 昇順
- `date-desc`: `parseDate(startDate)` 降順
- `title`: `localeCompare` でタイトル昇順

### 6.7 `CalendarView.vue`

**inject**: `openModal`  
**state**:

- `calYear` ref — `TODAY.getFullYear()` で初期化
- `calMonth` ref — `TODAY.getMonth()` で初期化（0-indexed）
- `selectedDay` ref — null（選択なし）または数値

**computed**:

- `firstDay` — 月初の曜日（0=日曜）
- `daysInMonth` — 当月の日数
- `isAtMaxMonth` — 現在月が TODAY + 6 ヶ月以上かどうかを判定（`calYear > maxCalYear || (calYear === maxCalYear && calMonth >= maxCalMonth)`）
- `isAtMinMonth` — 現在月が TODAY - 2 ヶ月以下かどうかを判定（`calYear < minCalYear || (calYear === minCalYear && calMonth <= minCalMonth)`）
- `monthEvents` — EVENTS のうち当月に重なるイベント（startDate <= 月末 && endDate >= 月初）、startDate 昇順ソート
- `displayEvents` — selectedDay が null なら monthEvents、そうでなければ選択日のイベント

**functions**:

- `eventsOnDay(d)` — monthEvents のうち選択日に重なるイベント
- `dayOfWeek(d)` — 曜日（0=日〜6=土）
- `isToday(d)` — 今日かどうか
- `isHoliday(year, month, d)` — data/holidays.js の `isHoliday` を利用
- `toggleDay(d)` — イベントが 0 件の日はクリックを無視する（`if (eventsOnDay(d).length === 0) return`）。selectedDay が d と同じなら null、そうでなければ d をセット。選択後はリストまでスムーズスクロール（50ms 後）
- `prevMonth()` / `nextMonth()` — 月移動（selectedDay をリセット）。prevMonth は `isAtMinMonth` が true のとき無効、nextMonth は `isAtMaxMonth` が true のとき無効。

### 7.2 ID 生成 `stableId(title, url)`

```js
const hash = crypto
  .createHash("md5")
  .update(title + url)
  .digest("hex");
return 10000 + (parseInt(hash.substring(0, 5), 16) % 89999);
```

同じタイトル・URL の組み合わせは常に同一 ID になる。

### 7.3 カテゴリ推定 `guessCategory(text)`

正規表現でテキストを分析し `{ category, label }` を返す。デフォルトは `{ category: 'culture', label: '文化・学習' }`。

| パターン                                                               | category   | label            |
| ---------------------------------------------------------------------- | ---------- | ---------------- |
| 工作・ものづくり・クラフト・アート・絵・版画・染め・陶芸               | experience | 体験・工作       |
| 展覧・展示・特別展・企画展・博物館・美術館                             | exhibition | 展覧会           |
| キャンプ・自然・昆虫・野鳥・観察・ハイキング・登山・川遊び・アウトドア | nature     | 自然・アウトドア |
| まつり・フェスタ・フェスティバル・祭り・縁日                           | festival   | 祭り・フェスタ   |
| それ以外                                                               | culture    | 文化・学習       |

### 7.7 バリデーション `isValidEvent(ev)`

以下の条件を満たさないイベントを除外する:

- `url` が未設定または `javascript:` を含む
- `title` が 8 文字未満
- `UI_TITLES` セットに含まれるタイトル
- 20 文字以下 かつ 館/課/センター/係/局/事務所/部 で終わるタイトル
- カテゴリ名のみのタイトル（正規表現で判定）

### 7.6 重複除去 `deduplicateEvents(events)`

ID をキーとした `Map` で重複を除去する（先着優先）。

### 7.9 手動イベント保持 `loadManualEvents()`

既存の `events.js` を読み込み、`eval` で EVENTS 配列を取得し、`id < 10000` のイベントのみ返す。  
返却時に `area` フィールドを正規化する（`利根郡みなかみ町` → `みなかみ町`等、郡名を除去）。
